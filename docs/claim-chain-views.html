What's rendered

Each mode takes the same set of ~20 core nodes from the claim-chain and relabels them for the view's idiom:

  • Max/MSP patch: primitives become number/signal objects ([int], [cycle~]), the mediant becomes [+~]/[/~] arithmetic, the circle map is a [phasor~] feedback patch, Klein-bottle identification is two [send~]/[receive~] pairs with opposite signs. Cords are yellow. Not a real .maxpat export — the format is JSON and could be emitted on request, but this is the network topology only.
  • Pure mathematics: nodes become expressions in LaTeX ($\theta_{n+1} = \theta_n + \Omega - (K/2\pi)\sin(2\pi\theta_n)$, $(a+c)/(b+d)$, $x = f(x)$, $x^2 + \mu = 0$). Edges are "uses lemma".
  • Syllogism: nodes become $(\text{premise}) \vdash (\text{conclusion})$ pairs, with edges labeled by the proof-chain proposition that bridges them (Proof A P1–P8, Proof B Q1–Q6). Best read top-down.

Layout

Nodes are laid out by topological depth: layer 0 are inputs (primitives and anchors), layer 1 are the composite objects built directly from them (mediant, circle map, Stern-Brocot tree), and so on up to the scorecard predictions at the deepest layers. Within a layer, nodes are sorted by id for stability.
